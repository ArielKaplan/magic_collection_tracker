A handoff document for new Claude sessions. Read this first before making changes.

## What this is

A **Windows desktop app** for tracking a Magic: The Gathering collection, with a primary focus on Secret Lair (SLD) drops. Built as an **Electron** shell wrapping a (mostly) vanilla HTML/JS/CSS renderer plus a **Svelte** free-form dashboard. Persistent local storage via **SQLite** (`better-sqlite3`).

Lives at: `C:\Users\Akapl\Documents\Secret Lair Tracker Desktop\`
Git: https://github.com/ArielKaplan/magic_collection_tracker (branch: `main`)
Current version: `0.2.0` (in `package.json`)

The user is a prolific MTG collector tracking thousands of cards (~4,700 entries, 6,000+ copies) across many binders. ManaBox CSV is the source of truth for imports.

There's an **older web app** at `C:\Users\Akapl\Documents\Secret Lair Tracker\` (HTML-only, no Electron). That's the predecessor — the desktop edition is the current one. Don't change the web app unless explicitly asked.

## Tech stack

| Layer | What |
|---|---|
| Shell | Electron 33 |
| Main process | Node.js, `better-sqlite3` for DB |
| Renderer (most tabs) | Vanilla HTML + CSS + JS — single file `src/renderer/app.js` (~3,200 LOC) |
| Renderer (Dashboard) | Svelte 4 components in `src/renderer-svelte/`, Vite-bundled to `src/renderer/dist/svelte-app.{js,css}` |
| Drag/resize | Interact.js |
| Build/packaging | electron-builder → NSIS installer |
| Storage | SQLite at `%APPDATA%\secret-lair-tracker\collection.db` |

External data sources:
- **Scryfall API** — card prices (`/cards/collection` POST endpoint, batched 75 IDs), card metadata (oracle text, type line, CMC, colors)
- **MTGJSON SLD.json** — Secret Lair drop → card mapping, scryfallId → drop subsets. Fetched via direct CORS, falling back to allorigins / corsproxy. Big file (~15 MB).
- **eBay Browse API** — sealed-product prices, user supplies their own OAuth credentials in Settings

## Tabs (in nav order)

1. **Dashboard** — Svelte free-form canvas with 18 drag-and-resize panels. Per-panel binder filter via the ⚲ icon. Layout persists in SQLite (`settings.dashboard_layout_v2`).
2. **My Collection** — Vanilla table view. Columns: name, set, foil, rarity, condition, lang, qty, cost, market, Δ price, trend sparkline, flags. Filters: binder sidebar, foil/rarity/condition/language. Search matches name + set + type_line + oracle_text.
3. **Sealed Collection** — Add/edit/track sealed products, eBay price lookups, manual pricing, sealed/opened status.
4. **Gallery** — Image grid of owned cards (Scryfall images). Search same as My Collection. **Hover** any card → preview popup.
5. **Secret Lair Explorer** — 3-level navigation: superdrops → drops → cards. Breadcrumb at top. Sort by date/name. Search by drop name OR card name. Per-tile owned/not-owned indicators (strict scryfallId match).
6. **Failed Lookups** — Cards where Scryfall pricing failed. Filter pills by reason. "Retry batch errors" button for rate-limit recovery.

Tabs are top-of-window underline-style; **renamed** from earlier "Cards", "Sealed Product", "Secret Lair" labels. Don't revert.

## Native chrome

- **Menu bar**: File / Edit / View / Tools / Help. Real keyboard accelerators: Ctrl+I import CSV, Ctrl+O load JSON, Ctrl+S save backup, F5 refresh prices, F11 fullscreen, Ctrl+1–6 switch tabs, Ctrl+, settings, **Ctrl+L toggle activity log**, Ctrl+Q quit.
- **Status bar (bottom)**: live card count · total value · last-refresh age · failure badge · `ⓘ Logs` button · autosave indicator.
- **Activity log panel**: slide-in from right, color-coded by level (info/success/warn/error), category-tagged (App, Price, SL, Import, Save). Wired into refreshes, imports, autoSave errors, startup.
- **Card hover preview**: 200ms-delay popup with image, name, set, price, type, oracle text. Works on Gallery tiles, My Collection rows, and SL Explorer tiles. Owned cards show full details; un-owned SL tiles show "Not in your collection" + drop info.

## Data flow

1. **Import CSV** (`File → Import CSV…` or Ctrl+I) — ManaBox export. Dedup on manaboxId + scryfallId + foil. Cards added to `collection.cards`, written to SQLite via `autoSave()`.
2. **Refresh Prices** (F5) — gathers unique (scryfallId, foil) pairs, batches 75-at-a-time to Scryfall, retries 429s with exponential backoff (2s/4s/8s), stores prices in `price_history` table. Also captures card_metadata (colors, type_line, cmc, oracle_text).
3. **Refresh SL Data** — fetches MTGJSON SLD.json (`json.data.cards` array, NOT `Object.values(json.data)` — this was a bug), builds scryfallId→drops + scryfallId→name maps, does a second-pass collector-number backfill for foils. Stored in `sl_drop_cards`, `sl_scryfall_drops`, and `settings.sl_scryfall_to_name`.

All long-running operations log to the activity log AND `console`.

## Known quirks / fragile areas

- **Scryfall IDs are normalized to lowercase** at every entry point (CSV import, JSON load, refresh). Old user data may have mixed-case IDs — a one-time migration runs on autoLoad.
- **Foil → etched price fallback** in `refreshPrices`: many SLD foils only have `usd_etched` not `usd_foil`. Without this fallback, hundreds of cards "fail" pricing. Don't remove.
- **MTGJSON `subsets` field is hand-curated** and patchy for foils/variants. The collector-number backfill catches most of these. If a user reports "I own X but it's marked unowned", first ask them to Refresh SL Data.
- **Superdrop hierarchy is hand-curated** in `secretlair.js` `SL_SUPERDROPS`. Filled in through 2025-11 (Secretversary 2025) + a 2026-01 placeholder. Drops not in any superdrop auto-bucket to "Recent Additions" after a refresh. There's a **planned curation UI** (Settings → Edit Secret Lair Superdrops) that hasn't been built yet.
- **PowerShell + git here-strings**: PowerShell can mis-parse `git commit -m @'...'@` when the body contains double-quoted phrases. Workaround: assign to a `$msg` variable first, then `git commit -m $msg`.
- **Native binaries**: `better-sqlite3` requires Electron-specific rebuild. If `npm install` ever fails, do `npm install --ignore-scripts` then `npx @electron/rebuild -f -w better-sqlite3` then `node node_modules/electron/install.js`.
- **Windows Developer Mode must be ON** for electron-builder to extract macOS-symlink files in its winCodeSign cache. Otherwise builds fail with "A required privilege is not held by the client".

## Working rules / user preferences

These are explicit user-stated preferences. Honor them.

- **Bump `package.json` version on every installer build.** Minor (`0.X.0`) for feature work, patch (`0.X.Y`) for fix-only builds. Mention which level in chat. Commit the bump as its own commit titled "Bump version to X.Y.Z".
- **Commit + push to `main` is the default** when user says "commit" or "make it official". Use `--no-ff` for feature-branch merges so they're visible in history.
- **Don't push without being asked** — pushing is a public action.
- **User runs Windows + PowerShell** with execution policy that blocks `.ps1` scripts. Always prefix shell calls with `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass -Force` and use `npm.cmd` / `npx.cmd` (the `.cmd` variants).
- **The PATH from prior PowerShell sessions can be stale**. After Node installs or similar, refresh with `$env:Path = [System.Environment]::GetEnvironmentVariable("Path","Machine") + ";" + [System.Environment]::GetEnvironmentVariable("Path","User")`.
- **Git config**:
  - `user.name = Ariel Kaplan`
  - `user.email = Ariel.Kaplan@ppfa.org`
- **Friend-sharing model**: user installs his own .exe on top of his existing dev install; data at `%APPDATA%\secret-lair-tracker\collection.db` persists across upgrades. Friend's install on his own PC starts with an empty DB. Don't bundle user data in the installer.

## Common tasks

```powershell
# Launch in dev (rebuilds Svelte bundle, then opens Electron)
cd "C:\Users\Akapl\Documents\Secret Lair Tracker Desktop"
npm start

# Build distributable Windows installer (output: dist\Secret Lair Tracker Setup X.Y.Z.exe)
npm run build

# Just rebuild the Svelte bundle (no Electron launch)
npm run build:renderer

# Commit + push (the typical workflow)
git add .
git commit -m "Short summary"
git push
```

When making a code change that affects:
- **A Svelte component** → must rebuild renderer (`npm run build:renderer`) before the change shows up. `npm start` does this automatically.
- **A main-process file (main.js, db.js, preload.js)** → must restart Electron (not just reload the renderer).
- **A vanilla renderer file (app.js, secretlair.js, styles.css, index.html)** → just close+reopen the app, OR press Ctrl+R inside the app (which reloads only the renderer).
